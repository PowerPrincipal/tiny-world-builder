## Durable Architecture Facts

**Source shape**
- Entry: `tiny-world-builder.html` — boot config, DOM scaffold, ordered `<script src>` tags
- Styles: `styles/tiny-world.css` (~5.2k lines)
- Modules: `engine/world/00-*.js` through `70-*.js` + `99-late-boot.js` (~40k LoC total)
- All modules are **plain classic scripts sharing one global scope** — load order matters; every top-level `const`/`let`/`function` must be unique across all files. Duplicate identifier → `SyntaxError` silently kills that module while others continue. Prefix module-local globals (e.g. `_sr`, `_fl`, `_la`). New modules expose via `window.__*` from an IIFE.
- `tools/check.js` (`npm test`) fails on duplicate top-level identifiers across `engine/world/*.js`
- Three.js **r185** pinned; vendor self-hosted under `vendor/three/tinyworld-three.r185.min.js`. Do not bump.
- No bundler, no npm runtime dependencies. Static checks: `npm test`. Dist: `npm run build` or `./publish.sh`

**World data model**
- `world[x][z]` — intent: `{ terrain, terrainFloors, kind, floors }`
- `cellMeshes['x,z']` — render: `{ tile: Group, object: Group|null }`
- Always mutate via `setCell(x, z, opts)` — never write `world[x][z]` directly outside init
- `userData.landing` guards drop-in animations — never remove these checks

**Materials**
- `M.*` materials are shared across many meshes — clone before mutating `.color`
- `disposeGroup(group)` disposes geometries but NOT materials; per-instance materials (smoke) self-dispose on death

**Cameras**
- `orthoCam`, `softCam`, `persCam` coexist; `camera` is a swappable reference
- `togglePerspective()` / `setCameraMode()` write to all projections/positions

**LandscapeEngine**
- `LandscapeEngine.js` is a superseded monolith — `getHeight`/chunk-building live in `engine/landscape/*.js` mixins that override it. Edit the mixins, not the monolith.

**Build pipeline**
- Dev: `npm run dev` (:3000) serves source directly — no build step during iteration
- Prod: `./publish.sh` → `dist/` → Netlify auto-deploys; `vercel.json` + `netlify.toml` both point to `dist/`
- PartyKit server (`party/index.js`) deploys via `partykit deploy`, NOT `publish.sh`

**Geometry caching (added ~2026-07-01):** Shared geometries are now cached; `a59161e`/`d10566c` added render guardrails and stamp diagnostics to reduce per-frame allocation at larger grid sizes.

**Economy — two currency rails, never conflate**
- **Allowance GOLD** — holdings-based, non-transferable weekly allowance; `gold_ledger_events` table
- **Earned GOLD (Coins)** — player-to-player transferable; earned via template sales, referrals, resource sell; `coin_balances`/`coin_ledger` tables; `lib/coins.mjs` credit/debit/transfer primitive (adversarially reviewed, advisory-lock race-safe)
- GOLD is non-withdrawable with no real-world value and no conversion path to $TINYWORLD — this is the premise that makes public farming safe; news item commit 5913a42 contradicts this (see open threads)

**Economy gate architecture**
- `requireTinyverseAccess(auth.user, origin)` in `lib/tinyverse-access.mjs` guards all SPEND surfaces after auth
- Allowlist widened without redeploy via `TINYVERSE_ACCESS_EMAILS` env var (comma-separated)
- Exception: `/api/admin/gold-payout` gated by `x-admin-secret` only — intentional admin path
- Exception: `stripe-webhook` gated by Stripe signature verification only
- EARN side (resources-sell) is ungated — any signed-in player can sell harvested resources
- `gold-spend.mjs` missing gate fixed commit 7c9c8d4
- PR #101 merged — Tinyverse access check updated to use trusted email source (`user.email` not `profiles.email`)

**Public world access rules**
- `kind=starter && status=published` worlds accessible to any signed-in player (play token issued)
- Non-starter, draft, and unpublished worlds remain invite-only or owner-only
- Anonymous visitors cannot observe or harvest — must be signed in
- Owner multiplier (2x) is server-derived from `auth.user.email`; not spoofable via body
- `worldDto.ownerEmail` is opt-in excluded by default (PII fix)

**$TINYWORLD on-chain**
- Token-2022 mint: `7bK4jRMa3aY85wJMQEQqWsmY9mmrRT32AFWZ23cBpump` (6 decimals, ~1B supply, pump.fun)

- `TINYWORLD_TOKEN_MINT` SET in Netlify env; `SOLANA_RPC_URL` still needs Helius/QuickNode (public RPC rate-limits)
- Holdings overlay on home page is EMPTY until players link wallets

**Multiplayer message routing trap**
- New world-room handlers go in `onWorldMessage`, NOT the main `onMessage` (world rooms early-return past it). Shipped chat-emotes emote relay dead in prod via this trap.

**Observer role capability (verified empirically)**
